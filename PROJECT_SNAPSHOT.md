# 🌌 AXIS - Project Snapshot (March 2026)

## 📌 Project Overview
**AXIS** is a high-performance, autonomous AI framework designed for spatial computing and complex system orchestration. It uses a hybrid **Plan-and-Execute** architecture, combining cloud-based strategic reasoning with secure, localized tool execution.

**Core Mission**: To provide a unified interface for managing local files, system operations, and external MCP services through a single "digital brain."

---

## 🛠️ Technology Stack
- **Core Engine**: Python 3.11+
- **Strategic Planner**: Google Gemini 2.0 Flash (Cloud Reasoning)
- **Tactical Executor**: Qwen 2.5 Coder 7B (Local via Ollama)
- **Vector Memory**: ChromaDB (Semantic Search & Caching)
- **Communications**: Telegram Bridge (Remote Control) & PyQt6 (Localized HUD)
- **Protocols**: MCP (Model Context Protocol), Secure WebSocket
- **Quality Control**: Ruff (Linter), Pytest (Auto-Verification)

---

## 🧠 Architectural Innovations (March 2026)

### 1. Plan-and-Execute Framework
The system is bifurcated into two specialized subprocesses:
- **Planner**: Uses High-Reasoning LLMs to break down complex requests into atomic execution plans.
- **Executor**: Uses local coding LLMs to execute tasks "silently" and efficiently using local tools.

### 2. Universal MCP Hub
AXIS is now a fully protocol-aware agent. It can connect to any external **Model Context Protocol** server (GitHub, Notion, Docker, etc.) via its internal `MCPBridge`, treating remote tools with the same ease as local Python functions.

### 3. Semantic Caching (RAG 2.0)
To reduce latency and CPU load (~30-40%), AXIS caches successful **Command -> Execution Plan** mappings in ChromaDB. Recurring commands bypass the Planner entirely, triggering instant "semantic instincts."

### 4. Shadow Workspace Verification
Safety-first approach for code modifications. AXIS creates a `.shadow_copy.py`, runs **Ruff** checks and **Pytest** verification, and only replaces the original file upon 100% success.

## 🧰 Available Agent Skills

### 📂 Files (file_master)
- `read_file`, `write_file`, `delete_file`, `list_dir`, `search_files`
- *Dynamic Context Feeding into RAG*

### 💻 System (terminal_operator, diagnostics, os_control)
- `execute_command`: Secure terminal access with self-healing ([CRITICAL SYSTEM DIRECTIVE]).
- `get_system_status`: Real-time telemetry (CPU/RAM/GPU).
- `take_screenshot`, `press_key`, `mouse_click`: Full OS-level GUI interaction.

### 🔌 Universal Bridge (mcp_hub)
- `call_mcp_tool`: Dynamic invocation of external MCP services.
- `hot_reload_skills`: Instant tool detection without system restart.

### 🧠 Intelligence (code_intelligence, memory_manager)
- `refactor_code`: Verified autonomous code generation with Shadow Copy.
- `verify_code`: On-demand system health check via Ruff.
- `store_fact` / `search_memory`: Semantic episodic memory.
- `find_code_usages`: Semantic symbol tracking across the project.

### 👁️ Media & Web (vision_eye, web_research)
- `analyze_image`: Spatial understanding via Vision Engine.
- `google_search` / `browse_url`: Real-time internet grounding.

### 📡 Remote Control (telegram_bridge)
- Full HUD control and status reporting via Telegram.
- Human-In-The-Loop (HITL) for critical actions.
